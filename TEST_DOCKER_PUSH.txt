# Test Docker Push Manually

## Test if Docker push works from Jenkins container

### Option 1: Test from your local machine first

1. Open terminal/PowerShell

2. Login to Docker Hub:
   ```bash
   docker login
   # Username: didou2505
   # Password: [your password]
   ```

3. Tag an existing image:
   ```bash
   docker tag didou2505/fakarni-user-service:latest didou2505/fakarni-user-service:test
   ```

4. Try to push:
   ```bash
   docker push didou2505/fakarni-user-service:test
   ```

5. If this works, the issue is with Jenkins container network

### Option 2: Test from Jenkins container

1. Find Jenkins container name:
   ```bash
   docker ps | grep jenkins
   ```

2. Enter Jenkins container:
   ```bash
   docker exec -it <jenkins-container-name> bash
   ```

3. Try to login:
   ```bash
   docker login -u didou2505 -p [your-password]
   ```

4. Try to push an image:
   ```bash
   docker push didou2505/fakarni-user-service:8
   ```

### If push fails with timeout:

This could be:
- Network connectivity issue
- Docker Hub rate limiting
- Firewall blocking Docker Hub
- VPN/Proxy issue

### Solution: Check Docker Hub Status

1. Go to: https://status.docker.com/
2. Check if Docker Hub is having issues

### Alternative: Use Docker Hub Access Token

Instead of password, use an access token:

1. Go to: https://hub.docker.com/settings/security
2. Click: "New Access Token"
3. Name: jenkins-push
4. Permissions: Read, Write, Delete
5. Generate
6. Copy the token
7. Update Jenkins credentials to use this token instead of password
